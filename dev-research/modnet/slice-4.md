# Slice 4

## Target

Define enterprise and shared-service node exchange patterns without collapsing
sovereignty or treaty boundaries.

## Scope

- modnet research lane only
- enterprise/shared-service node patterns
- treaty boundaries between personal, enterprise, and service nodes
- no collapse into a centralized host model

## Required

- distinguish personal-node and enterprise/shared-service-node exchange needs
- define how shared-service nodes expose capabilities without bypassing node
  sovereignty
- preserve A2A and PM authority across these patterns

## Preserve

- sovereign node boundaries remain explicit
- personal and enterprise nodes remain peers across treaty boundaries
- imported artifacts and services remain governed rather than implicitly trusted

## Avoid

- central platform assumptions
- service nodes acting like global mutable repos
- bypassing PM authority because a node is organizational

## Acceptance Criteria

- enterprise/shared-service exchange patterns are explicit
- the design keeps node sovereignty intact
- the resulting patterns stay compatible with the earlier modnet slices
