# Vertex Schemas

Complete JSON-LD shapes for all vertex types in the hypergraph memory.

This file mixes:
- **current concrete shapes** used by Plaited tooling today
- **architectural direction** for richer memory graphs

When they differ, the current code in `src/tools/` and `src/agent/` wins.

## SelectionDecision (Runtime Hyperedge)

One per BP superstep. The core hyperedge ŌĆö connects threads, events, and decisions.

```jsonc
// .memory/sessions/sess_abc/decisions/007.jsonld
{
  "@context": "../../../@context.jsonld",
  "@id": "session/sess_abc/decision/7",
  "@type": "SelectionDecision",
  "superstep": 7,
  "timestamp": "2026-03-03T10:00:00.123Z",
  "bids": [
    {
      "@type": "Bid",
      "event": "bp:event/execute",
      "thread": "bp:thread/sim_guard_tc-1",
      "selected": false,
      "blockedBy": "bp:thread/sim_guard_tc-1",
      "priority": 0
    },
    {
      "@type": "Bid",
      "event": "bp:event/tool_result",
      "thread": "bp:thread/batchCompletion",
      "selected": true,
      "priority": 0
    }
  ]
}
```

### Bid Fields

| Field | Type | Meaning |
|---|---|---|
| `event` | `@id` | Which event this bid proposes |
| `thread` | `@id` | Which thread proposed it |
| `selected` | boolean | Whether this bid won event selection |
| `blockedBy` | `@id` | Which thread blocked this event (if blocked) |
| `interrupts` | `@id` | Which thread was interrupted when selected |
| `priority` | integer | Thread priority (lower = higher priority) |

## Session (Summary Vertex)

Written at session completion. Contains embedding for similarity search.

```jsonc
// .memory/sessions/sess_abc/meta.jsonld
{
  "@id": "sess_abc",
  "@type": "Session",
  "summary": "Implemented file watcher with debounce. Gate rejected rm -rf attempt.",
  "embedding": [0.023, -0.118, 0.445, "..."],
  "threadTypes": ["taskGate", "maxIterations", "noRmRf"],
  "outcomeEvents": ["gate_rejected", "message"],
  "toolsUsed": ["read", "write", "bash"],
  "decisionCount": 12,
  "timestamp": "2026-03-03T10:00:00Z"
}
```

**Progressive disclosure:** The agent reads the tree of `meta.jsonld` files, decides relevance from summaries/embeddings, then loads specific session decision files.

## Commit (Training Anchor)

Links decisions to code changes. Written by `commit_snapshot` handler after each side-effect commit.

```jsonc
// .memory/sessions/sess_abc/commits/a1b2c3d.jsonld
{
  "@context": { "bp": "urn:bp:" },
  "@id": "bp:commit/a1b2c3d",
  "@type": "Commit",
  "sha": "a1b2c3d",
  "modulePath": "/abs/path/to/module",
  "attestsTo": [
    "session/sess_abc/decision/42",
    "session/sess_abc/decision/43"
  ],
  "timestamp": "2026-03-14T10:00:00Z"
}
```

**One-behind pattern:** The commit vertex for commit N lives in commit N+1 ŌĆö the SHA is only known after the commit completes. `attestsTo` links are correct; the vertex just appears in the next commit's tree.

**Training traversal:** `decision/42 ŌåÉ attestsTo ŌåÉ bp:commit/a1b2c3d`

## Skill (Design-Time Subgraph)

Points to the markdown source and currently captures light structural references.
Regenerated by `ingest-skill` when source changes.

```jsonc
// .memory/skills/behavioral-core.jsonld
{
  "@context": {
    "bp": "urn:bp:",
    "schema": "https://schema.org/"
  },
  "@id": "skill://behavioral-core",
  "@type": "Skill",
  "schema:name": "behavioral-core",
  "schema:description": "Behavioral programming patterns for coordination",
  "source": "/abs/path/to/skills/behavioral-core/SKILL.md",
  "timestamp": "2026-03-14T10:00:00Z",
  "provides": [
    {
      "@id": "bp:thread/taskGate",
      "@type": "Thread",
      "references": ["bp:event/task", "bp:event/execute", "bp:event/tool_result"]
    }
  ],
  "requires": []
}
```

Current implementation note:
- `ingest-skill` currently scans TypeScript references for `bp:thread/*`,
  `bp:event/*`, and simple cross-skill import dependencies.
- richer section/provenance encoding is now handled by the newer
  `skill-links` / `validate-encoding` tooling rather than the base
  `ingest-skill` vertex alone.

### CONTRACT ŌåÆ JSON-LD Mapping

| CONTRACT field | JSON-LD equivalent |
|---|---|
| `produces` | Candidate vertex definitions with `@type` and `@id` |
| `consumes` | `@id` references to vertices from other skills |
| `depends-on` | `requires` edges between skill vertices |
| `layer` / `wave` | Architectural metadata, not part of current `ingest-skill` output |

## RuleSet (AGENTS.md Subgraph)

Ingested from AGENTS.md files. Scope derived from file path.

```jsonc
// .memory/rules/workspace.jsonld
{
  "@context": "../@context.jsonld",
  "@id": "rules://project",
  "@type": "RuleSet",
  "source": "AGENTS.md",
  "scope": "project",
  "rules": [
    {
      "@id": "rule://no-index-files",
      "@type": "GovernanceRule",
      "schema:name": "Module Organization",
      "content": "No index.ts ŌĆö rename to feature name."
    },
    {
      "@id": "rule://use-test-not-it",
      "@type": "GovernanceRule",
      "schema:name": "Testing",
      "content": "Use test not it."
    }
  ]
}
```

### Scope Resolution

| File Path | Scope |
|---|---|
| `AGENTS.md` | `project` |
| `modules/apple-block/AGENTS.md` | `apple-block` |

Module-scoped rules overriding workspace rules is architectural intent; the
generic hypergraph tool itself does not implement rule resolution policy.

## Design-Time Ōåö Runtime Connection

The `@id` URIs create implicit links between layers:

| Design-Time Vertex | Runtime Reference |
|---|---|
| `bp:thread/taskGate` in `threads/taskGate.jsonld` | `"thread": "bp:thread/taskGate"` in `decisions/002.jsonld` |
| `bp:event/execute` in thread phases | `"event": "bp:event/execute"` in decision bids |
| `skill://behavioral-core` in `skills/` | Any thread vertex it `provides` appearing in runtime decisions |

This enables queries like:
- "Which skills' threads participated in this session?" ŌĆö follow `@id` from decision ŌåÆ thread ŌåÆ skill
- "Has this thread ever been blocked?" ŌĆö find decisions referencing the thread's `@id` where `blockedBy` is set
- "Which skills conflict?" ŌĆö merge subgraphs, check for cycles in combined blocks
