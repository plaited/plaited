# Structural IA Lineage

> Status: source synthesis. Use this page to preserve useful lineage without
> turning historical design language into runtime law.

## Useful Ideas

Structural IA remains valuable because it asks questions Plaited still needs:

- What is the unit of meaning in this domain?
- Which relationships matter between objects, people, and systems?
- Which mechanics change incentives or effort?
- Which boundaries determine who sees or can act?
- Which mental maps help humans move through an experience?

The useful shift is from visual reuse to functional analysis. Plaited should
preserve that shift.

## Actor-Era Reframing

The old authoring question was:

```text
Which reusable UI structure does this module fit into?
```

The Plaited question is:

```text
What actor-owned facts, services, policies, and provenance can support useful
local projections for different humans and agents?
```

Structural IA helps name the domain. It should not force the user to manually
break ordinary activity into object hierarchies before the system can help.

## Human Movement Through Digital Space

Generated UI does not remove the need for legible experiences. Humans still
need orientation, memory, affordances, and predictable consequences.

Plaited should optimize for:

- low-effort authoring from ordinary language and artifacts
- clear local projections tailored to the user's task
- visible authority boundaries before cross-node exposure
- continuity across recurring workflows without freezing one canonical UI
- auditability when agents reorganize information

## What Not To Import

Do not import the old S1-S8 scale ladder as a normative ontology.

Do not treat prebuilt UI blocks as the network contract.

Do not assume a non-technical user should understand or author Structural IA
decomposition directly.

Do keep the vocabulary for reviewing whether generated experiences preserve
meaning, orientation, incentives, and boundaries.
